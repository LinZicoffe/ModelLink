# .NET Framework WinForms 项目 — Inno Setup 安装包打包教程

## 前置条件

| 工具 | 用途 | 安装方式 |
|------|------|----------|
| .NET Framework 4.8 Developer Pack | 编译项目 | [微软官网下载](https://dotnet.microsoft.com/download/dotnet-framework/net48) |
| MSBuild v4.0.30319 | 构建工具 | 随 Developer Pack 安装，路径：`C:\Windows\Microsoft.NET\Framework\v4.0.30319\MSBuild.exe` |
| Inno Setup 6 | 制作安装包 | [官网下载](https://jrsoftware.org/isdl.php)，默认安装路径：`C:\Program Files (x86)\Inno Setup 6` |

> **注意**：旧版 .csproj 格式（ToolsVersion 14.0）不支持 `dotnet build`，必须使用 MSBuild。

---

## 第一步：Release 构建

打开命令行（CMD 或 Git Bash），进入项目根目录（`.sln` 文件所在目录），执行：

**CMD:**
```cmd
"C:\Windows\Microsoft.NET\Framework\v4.0.30319\MSBuild.exe" 你的项目.sln /p:Configuration=Release
```

**Git Bash:**
```bash
/c/Windows/Microsoft.NET/Framework/v4.0.30319/MSBuild.exe 你的项目.sln '-p:Configuration=Release'
```

构建成功后，输出文件在项目的 `bin\Release\` 目录下。安装包只需要以下文件：

| 需要打包 | 不需要 |
|----------|--------|
| `*.exe`（主程序） | `*.pdb`（调试符号） |
| `*.exe.config`（运行配置） | `*.vshost.*`（VS 调试文件） |
| `*.dll`（依赖库） | `*.xml`（DLL 文档注释） |

---

## 第二步：准备数据模板

安装包不应包含开发环境的用户数据、日志等运行时文件。需要在项目中创建一个 `DataTemplates\` 目录，放置干净的初始数据文件。

例如，用户列表应提供空模板：
```json
[]
```

> 如果程序首次运行会自动创建缺失的数据文件，这步可以跳过。

---

## 第三步：编写 Inno Setup 脚本

在项目根目录创建 `setup.iss` 文件，基本模板如下：

```iss
; === 应用信息 ===
#define MyAppName "你的应用名称"
#define MyAppVersion "1.0"
#define MyAppPublisher "你的公司名"
#define MyAppExeName "你的程序.exe"

[Setup]
AppId={{随机GUID}}
AppName={#MyAppName}
AppVersion={#MyAppVersion}
AppVerName={#MyAppName} {#MyAppVersion}
AppPublisher={#MyAppPublisher}

; 安装目录
DefaultDirName={autopf}\{#MyAppName}
DefaultGroupName={#MyAppName}

; 输出设置
OutputDir=Output
OutputBaseFilename={#MyAppName}_v{#MyAppVersion}
Compression=lzma2/ultra64
SolidCompression=yes

; 权限与架构
PrivilegesRequired=lowest
ArchitecturesAllowed=x64compatible x86
ArchitecturesInstallIn64BitMode=x64compatible

; 界面
WizardStyle=modern
UninstallDisplayIcon={app}\{#MyAppExeName}

[Languages]
Name: "chinesesimplified"; MessagesFile: "compiler:Languages\ChineseSimplified.isl"

[Tasks]
Name: "desktopicon"; Description: "创建桌面快捷方式"; GroupDescription: "附加图标:"

; === 安装文件 ===
[Files]
; 主程序
Source: "项目目录\bin\Release\你的程序.exe"; DestDir: "{app}"; Flags: ignoreversion
Source: "项目目录\bin\Release\你的程序.exe.config"; DestDir: "{app}"; Flags: ignoreversion

; 依赖 DLL
Source: "项目目录\bin\Release\依赖库.dll"; DestDir: "{app}"; Flags: ignoreversion

; 数据模板（如果需要）
Source: "DataTemplates\users.json"; DestDir: "{app}\Data"; Flags: ignoreversion

; === 快捷方式 ===
[Icons]
Name: "{group}\{#MyAppName}"; Filename: "{app}\{#MyAppExeName}"
Name: "{group}\卸载 {#MyAppName}"; Filename: "{uninstallexe}"
Name: "{autodesktop}\{#MyAppName}"; Filename: "{app}\{#MyAppExeName}"; Tasks: desktopicon

; === 安装完成后 ===
[Run]
Filename: "{app}\{#MyAppExeName}"; Description: "启动 {#MyAppName}"; Flags: nowait postinstall skipifsilent

; === 卸载清理 ===
[UninstallDelete]
Type: filesandordirs; Name: "{app}\Data"
```

**要点说明：**
- `AppId`：全局唯一标识，可用 [在线 GUID 生成器](https://guidgenerator.com/) 生成
- `OutputDir`：安装包输出目录，这里是项目根目录下的 `Output\`
- `OutputBaseFilename`：安装包文件名
- `Source`：相对于 `.iss` 文件的路径，指向 Release 构建输出
- `DestDir`：`{app}` 代表安装目录

---

## 第四步：编译安装包

Inno Setup 安装路径：`C:\Program Files (x86)\Inno Setup 6`

### 方式一：命令行编译

**CMD:**
```cmd
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" setup.iss
```

**Git Bash:**
```bash
"/c/Program Files (x86)/Inno Setup 6/ISCC.exe" setup.iss
```

### 方式二：图形界面编译

1. 双击打开 `setup.iss`（自动用 Inno Setup Compiler 打开）
2. 点击菜单 **Build → Compile**（或按 Ctrl+F9）

### 编译成功标志

```
Successful compile (X.XXX sec). Resulting Setup program filename is:
...\Output\你的应用名称_v1.0.exe
```

安装包在项目根目录的 `Output\` 文件夹中。

---

## 第五步：验证安装包

1. 双击运行生成的安装包
2. 按向导完成安装
3. 检查安装目录，确认：
   - 程序 exe 和依赖 DLL 存在
   - 数据文件为空模板（无真实用户数据）
   - 无调试文件（.pdb、.vshost.*）
   - 无运行时产生的日志和缓存

---

## 修改版本号

版本号在 `setup.iss` 文件顶部定义：

```iss
#define MyAppVersion "1.0"
```

修改此值后重新编译，安装包文件名自动更新。

## 常见问题

### Q: 编译报错 "Couldn't open include file ... ChineseSimplified.isl"

**原因**：Inno Setup 6 默认不包含简体中文语言包。

**解决**：下载语言文件到 Inno Setup 的 Languages 目录（需管理员权限）：

**CMD:**
```cmd
curl -o "C:\Program Files (x86)\Inno Setup 6\Languages\ChineseSimplified.isl" https://raw.githubusercontent.com/jrsoftware/issrc/main/Files/Languages/Unofficial/ChineseSimplified.isl
```

**Git Bash:**
```bash
curl -sL "https://raw.githubusercontent.com/jrsoftware/issrc/main/Files/Languages/Unofficial/ChineseSimplified.isl" -o "/c/Program Files (x86)/Inno Setup 6/Languages/ChineseSimplified.isl"
```

> 此操作只需执行一次。

### Q: 编译报错找不到 exe 文件

**原因**：未执行 Release 构建，或 `.iss` 中 Source 路径写错。

**解决**：
1. 先执行第一步 Release 构建
2. 检查 `setup.iss` 中 `Source` 路径与实际 `bin\Release\` 下的文件名是否一致

### Q: 安装后程序无法启动，提示缺少 .NET Framework

**原因**：目标电脑未安装对应版本的 .NET Framework。

**解决**：从 [微软官网](https://dotnet.microsoft.com/download/dotnet-framework/net48) 下载安装 .NET Framework 4.8 Runtime。

### Q: 如何设置程序图标？

在 `setup.iss` 的 `[Setup]` 段中添加：

```iss
SetupIconFile=你的图标.ico
```

图标文件需放在 `.iss` 同目录或指定相对路径。

## 一键打包命令

```bash
# 1. 进入项目目录
cd "你的项目路径"

# 2. Release 构建
/c/Windows/Microsoft.NET/Framework/v4.0.30319/MSBuild.exe 你的项目.sln '-p:Configuration=Release'

# 3. 编译安装包
"/c/Program Files (x86)/Inno Setup 6/ISCC.exe" setup.iss
```

安装包输出到 `Output\` 目录，三行命令搞定。
